# Phase E1: Gateway + MCP/HTTP — Agent Connectivity

## Context

Terrarium compiles worlds and generates entities, but external agents have NO way to connect. The only entry point is `app.handle_action()` — a Python API. We need MCP and HTTP protocol support so agents can connect and interact with the simulated world exactly like they would with real services.

## The Core Insight (from spec)

> "**Each service in the world is exposed as an MCP server.** The Adapter translates MCP tool calls into world actions."

This means: when a world has email + Stripe + tickets, the agent sees `email_send`, `email_read`, `stripe_refunds_create`, `tickets_create` as **MCP tools** or **HTTP endpoints**. The agent doesn't know it's talking to Terrarium — it thinks it's talking to real email/Stripe/tickets.

**When a new service pack is added, its tools are AUTOMATICALLY available** — the tools come from `ServiceSurface.operations` which are already generated by `to_mcp_tool()` / `to_http_route()` in `kernel/surface.py`.

## How It Actually Works

```
Agent (MCP client or HTTP client)
  │
  │  calls: email_send({from: "a@b.com", to: "c@d.com", subject: "Hi", body: "Hello"})
  │
  ▼
GATEWAY
  │  1. Receive protocol-specific request (MCP tool call or HTTP POST)
  │  2. Resolve tool_name → (service_id, action) via PackRegistry
  │  3. Record to ledger (GatewayRequestEntry)
  │  4. Delegate to app.handle_action(actor_id, service_id, action, input)
  │
  ▼
PIPELINE (7 steps — already works)
  │  permission → policy → budget → capability → responder → validation → commit
  │
  │  The RESPONDER step:
  │    - Finds pack via PackRegistry.get_pack_for_tool("email_send")
  │    - EmailPack handler runs: creates email entity, returns response
  │    - Produces ResponseProposal with state deltas
  │
  ▼
GATEWAY
  │  5. Receive pipeline result (response_body dict)
  │  6. Format as protocol-specific response (MCP TextContent or HTTP JSON)
  │  7. Record response to ledger
  │  8. Return to agent
  │
  ▼
Agent receives: {"email_id": "email-abc123", "status": "sent", "thread_id": "thread-xyz"}
```

**Key:** The Gateway knows NOTHING about email, Stripe, or tickets. It just:
- Discovers tools from PackRegistry (already exists in WorldResponderEngine)
- Translates protocol ↔ ActionContext
- Delegates to `app.handle_action()` (already works)
- Formats response

## Rules

1. Gateway is PURE protocol translation — ZERO business logic
2. Every request goes through `app.handle_action()` → 7-step pipeline
3. Tool manifests come from PackRegistry (already populated by WorldResponderEngine)
4. When a new pack is registered, its tools are automatically available — no gateway changes
5. Event bus + ledger record every gateway request
6. Clean protocol separation — MCP adapter and HTTP adapter are independent
7. No heuristics, no fallbacks, no bypasses
8. Gateway discovers tools from the SAME PackRegistry the Responder uses — single source of truth

## Implementation

### 1. Gateway Core (`terrarium/gateway/gateway.py`)

```python
"""Gateway — single entry/exit point for all agent communication.

PURE protocol translation. ZERO business logic.
Discovers tools from PackRegistry (same source as WorldResponderEngine).
Routes all requests through TerrariumApp.handle_action() → 7-step pipeline.
Records every request to Ledger (GatewayRequestEntry).
"""

from __future__ import annotations
import logging
import time
from typing import Any

from terrarium.core.types import ActorId
from terrarium.gateway.config import GatewayConfig

logger = logging.getLogger(__name__)


class Gateway:
    """Single entry/exit point for all agent protocol communication."""

    def __init__(self, app: Any, config: GatewayConfig) -> None:
        self._app = app
        self._config = config
        self._adapters: dict[str, Any] = {}
        self._tool_map: dict[str, tuple[str, str]] = {}  # tool_name → (service_id, action)
        self._started = False

    async def initialize(self) -> None:
        """Discover tools from packs and start protocol adapters."""
        # Build tool map from the SAME PackRegistry the Responder uses
        responder = self._app.registry.get("responder")
        if hasattr(responder, "_pack_registry"):
            pack_registry = responder._pack_registry
            for tool_info in pack_registry.list_tools():
                tool_name = tool_info["name"]
                service = tool_info.get("pack_name", tool_info.get("service", ""))
                self._tool_map[tool_name] = (service, tool_name)

        logger.info(
            "Gateway: discovered %d tools from %d packs",
            len(self._tool_map),
            len(set(s for s, _ in self._tool_map.values())),
        )

        # Also include tools from compiled world services (Tier 2)
        compiler = self._app.registry.get("world_compiler")
        # WorldPlan services are in compiler's last compiled plan
        # These are accessible via the resolved ServiceSurfaces

        # Start protocol adapters
        from terrarium.engines.adapter.protocols.mcp_server import MCPServerAdapter
        from terrarium.engines.adapter.protocols.http_rest import HTTPRestAdapter

        mcp_adapter = MCPServerAdapter(self)
        http_adapter = HTTPRestAdapter(self, self._config)
        self._adapters["mcp"] = mcp_adapter
        self._adapters["http"] = http_adapter

        self._started = True

    async def handle_request(
        self,
        protocol: str,
        actor_id: str,
        tool_name: str,
        arguments: dict[str, Any],
    ) -> dict[str, Any]:
        """Handle an incoming request from any protocol.

        1. Resolve tool → (service_id, action)
        2. Delegate to app.handle_action() → 7-step pipeline
        3. Record to ledger (GatewayRequestEntry)
        4. Return response

        This method contains ZERO business logic.
        """
        start = time.monotonic()

        # Resolve tool
        resolution = self._tool_map.get(tool_name)
        if resolution is None:
            # Capability gap — per spec, return structured response, not error
            await self._record_request(
                protocol, actor_id, tool_name, "capability_gap", 0,
            )
            return {
                "status": "capability_not_available",
                "message": f"Tool '{tool_name}' is not available in this world.",
                "available_tools": list(self._tool_map.keys()),
            }

        service_id, action = resolution

        # Delegate to pipeline (THE pipeline — no shortcuts)
        result = await self._app.handle_action(
            actor_id=actor_id,
            service_id=service_id,
            action=action,
            input_data=arguments,
        )

        latency_ms = (time.monotonic() - start) * 1000
        status = "error" if "error" in result else "success"
        await self._record_request(
            protocol, actor_id, tool_name, status, latency_ms,
        )

        return result

    async def get_tool_manifest(
        self, actor_id: str | None = None, protocol: str = "mcp",
    ) -> list[dict[str, Any]]:
        """Return tools available in the requested protocol format.

        Tools come from PackRegistry — the same source as WorldResponderEngine.
        When a new pack is added, its tools appear here automatically.

        TODO (E2): Filter by actor permissions/visibility.
        """
        responder = self._app.registry.get("responder")
        if not hasattr(responder, "_pack_registry"):
            return []

        pack_registry = responder._pack_registry
        tools: list[dict[str, Any]] = []

        for pack_meta in pack_registry.list_packs():
            pack_name = pack_meta["pack_name"]
            pack = pack_registry.get_pack(pack_name)
            from terrarium.kernel.surface import ServiceSurface
            surface = ServiceSurface.from_pack(pack)

            if protocol == "mcp":
                tools.extend(surface.get_mcp_tools())
            elif protocol == "http":
                tools.extend(surface.get_http_routes())
            elif protocol == "openai":
                tools.extend(op.to_openai_function() for op in surface.operations)
            elif protocol == "anthropic":
                tools.extend(op.to_anthropic_tool() for op in surface.operations)

        return tools

    async def _record_request(
        self, protocol: str, actor_id: str, tool_name: str,
        status: str, latency_ms: float,
    ) -> None:
        """Record gateway request to ledger."""
        ledger = self._app.ledger
        if ledger is None:
            return
        from terrarium.ledger.entries import GatewayRequestEntry
        entry = GatewayRequestEntry(
            protocol=protocol,
            actor_id=ActorId(actor_id),
            action=tool_name,
            response_status=status,
        )
        await ledger.append(entry)

    async def start_adapters(self) -> None:
        """Start all protocol adapter servers."""
        for name, adapter in self._adapters.items():
            await adapter.start_server()
            logger.info("Gateway: %s adapter started", name)

    async def shutdown(self) -> None:
        """Stop all protocol adapter servers."""
        for name, adapter in self._adapters.items():
            await adapter.stop_server()
        self._started = False
```

### 2. MCP Server Adapter (`engines/adapter/protocols/mcp_server.py`)

```python
"""MCP Server Adapter — exposes Terrarium world tools as MCP tools.

When an agent connects as an MCP client, it sees all service pack tools
(email_send, email_read, etc.) as MCP tools. Tool calls go through the
Gateway → Pipeline → Pack handler → Response.

Uses: mcp SDK (Server, stdio_server)
Uses: Gateway.handle_request() for all tool calls
Uses: Gateway.get_tool_manifest(protocol="mcp") for tool discovery
"""

from __future__ import annotations
import logging
from typing import Any

from terrarium.engines.adapter.protocols.base import ProtocolAdapter

logger = logging.getLogger(__name__)


class MCPServerAdapter(ProtocolAdapter):
    """Exposes Terrarium world as an MCP server."""

    protocol_name = "mcp"

    def __init__(self, gateway: Any) -> None:
        self._gateway = gateway
        self._server = None
        self._actor_id: str = "mcp-agent"  # Default, overridden by connection

    async def start_server(self) -> None:
        """Create MCP server with tool handlers."""
        from mcp.server import Server
        from mcp.types import TextContent, Tool

        server = Server("terrarium-world")

        @server.list_tools()
        async def list_tools() -> list[Tool]:
            """Return all tools available in this world."""
            raw_tools = await self._gateway.get_tool_manifest(
                actor_id=self._actor_id, protocol="mcp"
            )
            return [Tool(**t) for t in raw_tools]

        @server.call_tool()
        async def call_tool(name: str, arguments: dict) -> list[TextContent]:
            """Handle tool call → Gateway → Pipeline → Response."""
            import json
            result = await self._gateway.handle_request(
                protocol="mcp",
                actor_id=self._actor_id,
                tool_name=name,
                arguments=arguments or {},
            )
            return [TextContent(
                type="text",
                text=json.dumps(result, default=str),
            )]

        self._server = server
        logger.info("MCP server created with %d tools",
                     len(await self._gateway.get_tool_manifest(protocol="mcp")))

    async def run_stdio(self) -> None:
        """Run MCP server on stdio transport (for local agents)."""
        if not self._server:
            await self.start_server()
        from mcp.server.stdio import stdio_server
        async with stdio_server() as (read, write):
            await self._server.run(read, write, self._server.create_initialization_options())

    async def stop_server(self) -> None:
        """Stop the MCP server."""
        self._server = None

    def translate_inbound(self, raw_request: Any) -> Any:
        """Not used — MCP SDK handles protocol translation."""
        pass

    def translate_outbound(self, result: Any) -> Any:
        """Not used — MCP SDK handles protocol translation."""
        pass

    async def get_tool_manifest(self) -> list[dict]:
        """Delegate to gateway."""
        return await self._gateway.get_tool_manifest(protocol="mcp")
```

### 3. HTTP REST Adapter (`engines/adapter/protocols/http_rest.py`)

```python
"""HTTP REST Adapter — exposes Terrarium world tools as REST API endpoints.

Routes:
  GET  /api/v1/tools              — list available tools
  POST /api/v1/actions/{tool}     — execute a tool call
  GET  /api/v1/health             — health check
  GET  /api/v1/entities/{type}    — query entities (read-only)

Uses: FastAPI + Uvicorn
Uses: Gateway.handle_request() for all tool calls
"""

from __future__ import annotations
import asyncio
import logging
from typing import Any

from terrarium.engines.adapter.protocols.base import ProtocolAdapter

logger = logging.getLogger(__name__)


class HTTPRestAdapter(ProtocolAdapter):
    """Exposes Terrarium world as a REST API."""

    protocol_name = "http"

    def __init__(self, gateway: Any, config: Any = None) -> None:
        self._gateway = gateway
        self._config = config
        self._app_instance = None
        self._server_task = None

    async def start_server(self) -> None:
        """Create and start FastAPI server."""
        from fastapi import FastAPI, Query
        from pydantic import BaseModel

        app = FastAPI(
            title="Terrarium World API",
            description="Simulated world services — agents interact here",
        )

        class ToolCallRequest(BaseModel):
            actor_id: str = "http-agent"
            arguments: dict[str, Any] = {}

        @app.get("/api/v1/tools")
        async def list_tools(actor_id: str = Query(default="http-agent")):
            """List all tools available in this world."""
            return await self._gateway.get_tool_manifest(
                actor_id=actor_id, protocol="http"
            )

        @app.post("/api/v1/actions/{tool_name}")
        async def call_tool(tool_name: str, body: ToolCallRequest):
            """Execute a tool call through the full pipeline."""
            return await self._gateway.handle_request(
                protocol="http",
                actor_id=body.actor_id,
                tool_name=tool_name,
                arguments=body.arguments,
            )

        @app.get("/api/v1/health")
        async def health():
            return {"status": "ok"}

        @app.get("/api/v1/entities/{entity_type}")
        async def query_entities(entity_type: str):
            """Query entities from state (read-only view)."""
            state = self._gateway._app.registry.get("state")
            entities = await state.query_entities(entity_type)
            return {"entity_type": entity_type, "count": len(entities), "entities": entities}

        self._app_instance = app
        logger.info("HTTP REST adapter created")

    async def run_server(self, host: str = "0.0.0.0", port: int = 8080) -> None:
        """Run the FastAPI server (blocking)."""
        import uvicorn
        config = uvicorn.Config(self._app_instance, host=host, port=port, log_level="info")
        server = uvicorn.Server(config)
        await server.serve()

    async def stop_server(self) -> None:
        """Stop the HTTP server."""
        self._app_instance = None

    def translate_inbound(self, raw_request: Any) -> Any:
        """Not used — FastAPI handles request parsing."""
        pass

    def translate_outbound(self, result: Any) -> Any:
        """Not used — FastAPI handles response serialization."""
        pass

    async def get_tool_manifest(self) -> list[dict]:
        """Delegate to gateway."""
        return await self._gateway.get_tool_manifest(protocol="http")
```

### 6. Tool Manifest Generator (`engines/adapter/tool_manifest.py`)

```python
class ToolManifestGenerator:
    """Generates protocol-specific tool manifests from PackRegistry.

    This is the bridge: PackRegistry (discovered packs) → protocol-specific tool lists.
    When a new pack is added to PackRegistry, its tools automatically appear.
    """

    def __init__(self, pack_registry: Any) -> None:
        self._pack_registry = pack_registry

    def generate(self, protocol: str = "mcp", actor_id: str | None = None) -> list[dict]:
        """Generate tool manifest for the given protocol.

        Uses ServiceSurface.to_mcp_tool() / to_http_route() etc.
        """
        from terrarium.kernel.surface import ServiceSurface
        tools = []
        for pack_meta in self._pack_registry.list_packs():
            pack = self._pack_registry.get_pack(pack_meta["pack_name"])
            surface = ServiceSurface.from_pack(pack)
            if protocol == "mcp":
                tools.extend(surface.get_mcp_tools())
            elif protocol == "http":
                tools.extend(surface.get_http_routes())
            elif protocol == "openai":
                tools.extend(op.to_openai_function() for op in surface.operations)
            elif protocol == "anthropic":
                tools.extend(op.to_anthropic_tool() for op in surface.operations)
        return tools
```

## Why This Scales Automatically

When someone creates a new pack (e.g., `TicketsPack`):
1. Pack is registered in `PackRegistry` during Responder's `_on_initialize()`
2. Gateway discovers it via `pack_registry.list_tools()`
3. MCP adapter exposes its tools via `ServiceSurface.get_mcp_tools()`
4. HTTP adapter exposes its routes via `ServiceSurface.get_http_routes()`
5. **Zero gateway code changes needed**

The framework is: `Pack → PackRegistry → ServiceSurface → Protocol adapter → Agent`

## Additional Requirements

### Real-World API Paths — Pack-Owned, Not Gateway-Hardcoded

The central registry + tool discovery are still the single source of truth:
- `GET /api/v1/tools` — returns ALL tools from PackRegistry (discovery endpoint)
- MCP — agents call by tool name (`email_send`) — from registry
- HTTP — agents call via real-world paths — but paths come FROM THE PACK, not hardcoded in gateway

Flow:
```
Pack defines (in tool definition):
  name="email_send", http_path="/email/v1/messages/send", http_method="POST"
                    ↓
ServiceSurface.get_http_routes() returns {method, path, tool_name}
                    ↓
HTTP adapter auto-mounts: POST /email/v1/messages/send → resolves to email_send
                    ↓
gateway.handle_request(tool_name="email_send") → pipeline
```

**Zero hardcoding in gateway.** Pack owns its paths. Gateway reads them via `get_http_routes()`.

The `APIOperation.http_path` and `http_method` fields ALREADY EXIST on the model but are currently empty in EmailPack. We populate them in the pack's tool definitions.

**When a new pack is added** with proper http_path values, the HTTP adapter discovers and mounts its routes automatically — no gateway changes needed.

**Three layers of access:**
1. `GET /api/v1/tools` — discovery, shows ALL tools with names + paths + schemas
2. MCP tool call — by tool name (e.g., `email_send`)
3. HTTP call — by real-world path (e.g., `POST /email/v1/messages/send`)

All three resolve to the same pipeline execution.

### Event Streaming via WebSocket

For live dashboard / frontend event visualization:

```
WS /api/v1/events/stream    — WebSocket endpoint
  → Subscribes to EventBus
  → Streams events to frontend in real-time
  → Supports filtering by event_type, actor_id, entity_type
```

The EventBus already supports `subscribe()` with callbacks. The WebSocket adapter subscribes to `"*"` (all events) and pushes them to connected clients.

```python
# In http_rest.py, add WebSocket endpoint:
@app.websocket("/api/v1/events/stream")
async def event_stream(websocket: WebSocket):
    await websocket.accept()
    # Subscribe to EventBus
    async def on_event(event):
        await websocket.send_json(event.model_dump(mode="json"))
    bus = gateway._app.bus
    await bus.subscribe("*", on_event)
    try:
        while True:
            await websocket.receive_text()  # Keep alive
    except WebSocketDisconnect:
        await bus.unsubscribe("*", on_event)
```

This enables:
- Live event dashboard (frontend subscribes via WS)
- Real-time monitoring of agent actions
- Event replay (reconnect with `from_sequence` parameter)

## Context for Subagents

**What is the Gateway?** The Gateway is the SINGLE entry/exit point for ALL external agent communication with a Terrarium world. It is PURE protocol translation — it converts MCP tool calls or HTTP requests into internal `ActionContext` objects and routes them through the existing 7-step pipeline (`app.handle_action()`). It has ZERO business logic. It doesn't know what email or Stripe are — it just discovers tools from the PackRegistry and translates protocols.

**Why does it exist?** Without the Gateway, only Python code can interact with Terrarium (via `app.handle_action()`). With the Gateway, real AI agents can connect via MCP or HTTP and interact with the simulated world exactly like they'd interact with real services. An agent connecting to Terrarium's email service should see the SAME API shape as connecting to real Gmail.

**How does it fit in the architecture?**
```
Agent → Gateway (protocol translation) → app.handle_action() → Pipeline (7 steps) → Pack handler → Response → Gateway → Agent
```
The Gateway sits BETWEEN external agents and the existing pipeline. It doesn't replace anything — it wraps `app.handle_action()` with protocol support.

**What frameworks to REUSE (not rebuild):**
- `PackRegistry` (packs/registry.py) — discovers all packs and their tools
- `ServiceSurface.get_mcp_tools()` / `get_http_routes()` (kernel/surface.py) — converts pack tools to protocol formats
- `TerrariumApp.handle_action()` (app.py) — executes through the 7-step pipeline
- `GatewayRequestEntry` (ledger/entries.py) — ledger entry type (exists, never used)
- `CapabilityGapEvent` (core/events.py) — event for unknown tools
- MCP SDK (`mcp>=1.0`) — for MCP server
- FastAPI + Uvicorn — for HTTP REST server

**Rules for subagents:**
1. Gateway has ZERO business logic — all logic is in the pipeline
2. Every request goes through `app.handle_action()` — no shortcuts
3. Tools are discovered from PackRegistry — no hardcoding
4. HTTP paths come from pack definitions (`APIOperation.http_path`) — not gateway
5. Event bus + ledger record every gateway request
6. No heuristics, no fallbacks, no bypasses
7. Tests must use REAL protocol clients (MCP client SDK, httpx for HTTP) — not mock
